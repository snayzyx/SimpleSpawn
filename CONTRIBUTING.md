# Contributing to SimplesSpawn Plugin

Thank you for your interest in contributing to the SimplesSpawn Plugin! We welcome all contributions, including bug reports, feature requests, and code improvements.

## Getting Started

1. **Fork** the repository
2. **Clone** your forked repository
   ```bash
   git clone https://github.com/YOUR_USERNAME/Spawn.git
   ```
3. **Create a new branch** for your feature or bugfix
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Setup

### Prerequisites
- Java 17 or higher
- Gradle 8.0+
- Git

### Building the Project
```bash
./gradlew build
```

The compiled JAR will be available in `build/libs/`.

## Code Style

- Follow Java naming conventions (camelCase for variables/methods, PascalCase for classes)
- Use meaningful variable and method names
- Add comments for complex logic
- Keep methods focused and concise
- Use 4 spaces for indentation

## Making Changes

1. **Create a descriptive commit message**
   ```
   git commit -m "Add feature: [short description]"
   ```

2. **Keep commits atomic** - one feature or fix per commit

3. **Push to your branch**
   ```
   git push origin feature/your-feature-name
   ```

4. **Create a Pull Request** with a clear description of your changes

## Pull Request Guidelines

- Provide a clear description of the changes
- Link any related issues
- Ensure your code builds without errors
- Add comments for new features
- Update the README.md if necessary

## Reporting Issues

When reporting a bug, please include:
- Minecraft server version
- Plugin version
- Detailed steps to reproduce
- Error logs if applicable
- Your configuration file (without sensitive data)

## Feature Requests

When proposing a new feature:
- Describe the feature clearly
- Explain the use case
- Consider backward compatibility

## Code Review Process

All pull requests will be reviewed. We may request changes before merging.

## Questions?

Feel free to open an issue for any questions or contact the maintainers directly.
